# MUD Game (курсовой проект по ООП)

## Описание
Учебное **JavaFX**‑приложение в жанре **MUD (текстово‑приключенческая игра)** с простой 2D‑визуализацией на `Canvas`.  
Игрок попадает в локацию, может **перемещаться**, **атаковать врага (NPC)** и **подбирать предметы** (в том числе оружие, усиливающее атаку). Состояние игрока (здоровье) и инвентарь отображаются в интерфейсе.

## Технологический стек
- **Java**: 17+ (совместимый)
- **GUI**: JavaFX (Controls + Graphics, отрисовка на `Canvas`)
- **Сборка/запуск**: из IDE (есть Maven Wrapper, но `pom.xml` в текущем проекте пустой)
- **Тип приложения**: Desktop/Standalone GUI

## Статус
Учебный демо‑проект (**Beta**).

## Демо
Локальный запуск (см. раздел «Установка и запуск»).

## Архитектура
Проект разделён на:
- **UI/приложение**: класс `com.mudgame.MUDGame` — создаёт интерфейс JavaFX, обрабатывает кнопки, рисует игровое состояние на `Canvas`.
- **Модель**: пакет `com.mudgame.model` — классы предметной области игры (игрок, NPC, комната, предметы).

Основные сущности (папка `src/main/java`):
- **`com.mudgame.MUDGame`** — точка входа и GUI:
  - окно `800x600`, холст `Canvas 500x350`;
  - кнопки: «Войти в локацию», «Атаковать», «Переместиться», «Поднять предмет»;
  - инвентарь (`ListView`) и здоровье (`Label`);
  - размещение объектов в комнате и их перерисовка.
- **`com.mudgame.model.TGameObject`** — базовый абстрактный класс игровых объектов (имя, id, координаты `x/y`).
- **`com.mudgame.model.TRoom`** — локация/комната: хранит предметы и NPC, задаёт границы для отрисовки и перемещения.
- **`com.mudgame.model.TPlayer`** — игрок:
  - здоровье, базовая сила атаки, инвентарь;
  - действия: `attack(TNPC)`, `move()`, `pickUpItem()`;
  - общая сила атаки = базовая + сумма `TWeapon` в инвентаре.
- **`com.mudgame.model.TNPC`** — враг/NPC:
  - здоровье, защита, случайное перемещение по комнате `moveRandom()`;
  - получение урона `takeDamage(int)`, состояние `isAlive()`.
- **`com.mudgame.model.TItem`** — предмет (может лежать в комнате/быть поднят).
- **`com.mudgame.model.TWeapon`** — оружие (предмет с параметром `attackPower`, увеличивает атаку игрока).

## Зависимости
- **JDK**: 17+ (совместимый)
- **JavaFX**: модули `javafx.controls`, `javafx.graphics` (и `javafx.fxml` указан в `module-info.java`)
- **Сторонние библиотеки**: не используются
- **Media/звук**: не требуется

## Установка и запуск
Запуск из IDE (рекомендуется для текущего состояния проекта)
1. Откройте проект в IDE (например, IntelliJ IDEA).
2. Убедитесь, что выбран **JDK 17+**.
3. Настройте **JavaFX** в IDE (подключите JavaFX SDK / зависимости, чтобы модули `javafx.*` были доступны).
4. Запустите класс:
   - `src/main/java/com/mudgame/MUDGame.java` (метод `main`)



## Конфигурация
Специальной конфигурации нет: все параметры задаются в коде.

Полезные места:
- `src/main/java/com/mudgame/MUDGame.java` — UI, отрисовка, обработка кнопок, создание объектов
- `src/main/java/com/mudgame/model/*` — модель и игровые правила
- `src/main/java/module-info.java` — описание модулей Java 9+

## Применение
После запуска игры:
- Нажмите **«Войти в локацию»** — активируются действия, объекты размещаются в пределах комнаты и рисуются на холсте.
- Нажмите **«Переместиться»** — игрок переместится в случайную точку комнаты; NPC также может переместиться.
- Нажмите **«Атаковать»** — игрок наносит урон NPC.
  - урон рассчитывается как \(attackPower - defense\), минимум 1.
- Нажмите **«Поднять предмет»** — первый доступный предмет из комнаты добавится в инвентарь.
  - если поднято оружие, оно увеличивает суммарную атаку игрока (видно в списке инвентаря).

## Проверка ПО
Автоматические тесты не добавлялись (учебный проект). Ручные сценарии:
- запуск приложения;
- вход в локацию;
- перемещение (проверить перерисовку и перемещение NPC);
- атака NPC до “смерти” (NPC перестаёт рисоваться);
- подбор предмета и отображение в инвентаре;
- подбор оружия и проверка, что в инвентаре показывается бонус атаки.

## Проблемы
- Если игра не запускается, чаще всего причина — **не подключён JavaFX** или выбран неподходящий JDK.
